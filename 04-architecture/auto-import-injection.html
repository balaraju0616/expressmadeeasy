Auto Import Injection

How extensions automatically update app.js and end-points.js

What It Does

When you run StartEndPoint, AddSubRoute, or AddTableName, the extension does not just create files. It reads your existing app.js or routes.js and automatically injects the correct import and app.use() lines.

// Before running StartEndPoint:
// app.js has no Api import

// After:
import { router as apiRouter } from './Api/routes.js';
app.use('/Api', apiRouter);

How It Works Internally

The kschema-api-gen-actions module uses updateImports.js and updateUse/index.js to locate the correct insertion position, check for duplicates using checkDuplicate.js, then write the new lines safely.

cli.js
 ↓ parseInput.js
 ↓ resolveCommand.js
 ↓ updateImports.js + checkDuplicate.js
 ↓ updateUse/index.js
 ↓ Source file updated

Why process.cwd() Matters

All generators use process.cwd() to locate your project root. Running the command from the wrong folder generates incorrect paths and broken imports. Always run from your project root.

# Wrong (run from subfolder)
cd Api && npx @keshavsoft/kschema-api-gen-actions ShowAll

# Correct
npx @keshavsoft/kschema-api-gen-actions ShowAll
